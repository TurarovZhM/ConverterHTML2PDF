ParallelReadingOfSignalFile

communication with running MESHFREE-job by separate (parallel) thread

MESHFREE starts a separate thread, that mostly sleeps, but once in a second, it checks if the contents of the SIGNAL-file has changed. If changed, it triggers the appropriate actions by signal handlers. Advantage of this is, that MESHFREE will not have to pause in order to interprete the SIGNAL-file. The parallel processing of the SIGNAL-file is invoked ONLY in the common_variables.dat by the option
SIGNAL_LaunchComputationalSteering = true
List of members:

pause_on let MESHFREE sleep until the pause_off command is launched
