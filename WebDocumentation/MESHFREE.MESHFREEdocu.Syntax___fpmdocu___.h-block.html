h-block

h=hierarchy; short name of hierarchy element in the context of the whole hierarchy tree

The hierarchy defintion has to be enclosed in "#h" and "h#". Camel case is okay, but blanks are not allowed. A hierarchy level definition usually looks like this:
!$FPMDOCu #h Level1.Level2.Level3.NameOfTheCurrentLevel h# #a Optional alias statement a#
For instance the definition of this particular page is "!$FPMDOCu #h MESHFREE.MESHFREEdocu.Syntax___fpmdocu___.h-block h#" . Subsequent hierarchy definition: the statement
!$FPMDOCu #h ..NameOfTheCurrentLevel h#
will attach the given hierarchy element as a child to the last defined element, hence "!$FPMDOCu #h MESHFREE.MESHFREEdocu.Syntax___fpmdocu___.h-block h#" . !$FPMDOCu #h ..Child h# would define a hierarchy level with the name "MESHFREE.MESHFREEdocu.Syntax___fpmdocu___.h-block.Child" Hierarchy element in the same level: the statement
!$FPMDOCu #h .NameOfTheCurrentLevel h#
will attach the hierarchy element as a new member in the same row as the last defined element, hence the definitions "!$FPMDOCu #h MESHFREE.MESHFREEdocu.Syntax___fpmdocu___.h-block h#" . "!$FPMDOCu #h .another-block h#" . would define a hierarchy level with the name "MESHFREE.MESHFREEdocu.Syntax___fpmdocu___.another-block" Using text for more than one hierarchy items, one can define a list of items, separated by "||":
!$FPMDOCu #h MESHFREE.Solver.Sol1 || MESHFREE.Solver.Sol2 || MESHFREE.Solver.Sol3 || MESHFREE.AndSoOn h# #b !$FPMDOCu Now some text, that refers to all of the hierarchy items MESHFREE.Solver.Sol1, MESHFREE.Solver.Sol2, MESHFREE.Solver.Sol3, MESHFREE.AndSoOn !$FPMDOCu b#
Important is the a-block ("#a...a#"), which allows a onle-line-description of the hierarchy item. With the link-command ("#link"), the html-generator even tries to find the pages, which reference to the present item. Thus, an extension of the example above would be !$FPMDOCu #h Level1.Level2.Level3.NameOfTheCurrentLevel h# #a Optional alias statement a# #link With the sort-command ("#sort"), the html-generator will sort the list of children of this element before displaying the list of children in the html-output Thus, an extension of the example above would be !$FPMDOCu #h Level1.Level2.Level3.NameOfTheCurrentLevel h# #a Optional alias statement a# #link #sort
