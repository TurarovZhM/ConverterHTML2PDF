TestManagement

Test Management Plan

Testing Cycle

  • Every few hours*: Compilation of current master on local sytem. Release executable runs through beta tests.

  • Nightly*: Additional compilation of current master on multiple systems for client versions.

  • Every 2 months: Every new beta version runs through stable tests. No manual checks.

  • Biannual: Latest beta version becomes stable candidate and runs through the stable tests. Results are checked manually. If all tests are successful, this becomes the new stable version.

*) if there have been changes in the source code

Test Suite

  • Beta tests

    • 43 test setups.

    • Portion of OnlineExamples and further small examples.

    • Run time per test no more than 4 hours.

    • automatic detection of success/failure via INTEGRATION and EVENT statements.

  • Stable tests

    • 11 additional setups.

    • Large setups from industrial applications, including setups from clients.

    • Some automatic detection of failures via INTEGRATION and EVENT statements.

    • For many of these tests, manual examination is required, before they are marked as successful.

Recent testing improvements

  • Switched format for test setups to allow for more flexibility.

  • Added a large portion of tutorial setups and some other OnlineExamples to the test suite.

  • Created a dashboard with overview of test results.

Planned testing improvements

  • Add regular testing of debug executable with (possibly shortened) beta tests.

  • Add regular testing of debug executable with shortened stable tests.

  • Add regular and automatic testing of determinism in MPI mode.

  • Add remaining OnlineExamples to test suite.

  • Ongoing

    • Extend existing test setups with more automatic checks for problems.

    • Extend test suite with any new OnlineExamples added to the documentation.

    • Extend test suite with setups from projects and support tickets.
