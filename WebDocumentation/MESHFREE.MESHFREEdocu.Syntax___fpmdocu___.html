Syntax

syntax of the MESHFREEdocu statements

MESHFREEdocu is the way features of MESHFREE are documented. The documentation can be placed:
  • withing the code itself (in *.f90) files
  • in separate *.fpmdocu files

External and internal documentation

There is an internal and an external view onto the docu.
  • external: as it is uploaded to the server for customers
  • internal: external + additional documentation only visible for MESHFREE developers.
Where the documentation is placed in code (*.f90) files it has to be commented by
  • external: '!$FPMDOCu'
  • internal: '!$FPMINTDOCu'

Syntax

A documentation page consists at least of the following parts (in that order):
  2. Hierarchy definition, providing a hierachy connection given by a short name of the hierarchy element, see h-block.
  3. Subtitle definition: expanding the short hierarchy name into a compact, understandable sentence or phrase, see a-block.
  4. Text block: content of the documentation: see t-block, b-block and p-block.
The pages can contain:
  • Headings
  • Tables, see Table-block
  • frame-block
  • Ucv-block #UCVCODE (TODO: documented!)
  • References, see ref-block
  • images, see image-block
  • Latex equations, see latex-equation and latex-equation-inline
Automatic linking features, see
  • sort-command
  • link-command
List of members:
a-block a=alias; short description of the hierarchy element in one line
b-block b=block; full explanation of the hierarchy item in text-block from
BlindOutSubstring blinds out a marked substring such that it is not rendered in the html-document
frame-block write the lines in a box
h-block h=hierarchy; short name of hierarchy element in the context of the whole hierarchy tree
Headings Headings for structuring
image-block include a picture in the text
latex-equation translates LaTeX into pictures
latex-equation-inline translates LaTeX into pictures
link-command find list of pages/items, which reference to the present page/item
p-block p=passage; automates line breaks
ref-block invoke a reference link in the html-output
sort-command sort the list of children in lexicographical order in the html-output
t-block t=text; full explanation of the hierarchy item
Table-block write the lines into a table
TextDecoration give colors of emphasis on the text
void-block suppress the printing of the whole text body inside of the html-file
