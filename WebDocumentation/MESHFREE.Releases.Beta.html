Beta

Release notes for the MESHFREE beta executables

WARNING: beta versions -- reduced test suite only

Executables available at
https://svn.itwm.fraunhofer.de/svn/MESHFREEdocu/Executables/MESHFREE/beta/release/

If you would like to be notified about new beta versions of MESHFREE, please subscribe to the MESHFREE-Beta mailing list.

beta2020.06.0
New:
  • SCAN_ClustersOfConnectivity can be defined chamberwise.
  • Changed default for LINEQN_solver to 'BCX2'.
  • Changed default for LINEQN_solver_ScalarSystems to 'BCN2'.
  • DROPLETPHASE: %ind_Vi% now contains the particle volume for dropletphase chamber.
  • DROPLETPHASE: Adhesive/cohesive potential for DropletCollisions is now wrt. particle mass.
  • DROPLETPHASE: new Ucv/cv parameter DP_UseOnlyRepulsiveContactForce.
  • DROPLETPHASE: %BND_COLLISION%: Parameter SplitFactor and new index ind_diss_BE for storing by the BE dissipated energy.
  • DROPLETPHASE: %BND_COLLISION%: Parameter theta for modelling wall roughness.
  • DROPLETPHASE: parameters COEFF_dt_d30, COMP_DropletphaseSubcycles also in Ucv.
  • DROPLETPHASE: New example CollidingDropletsInCone.
  • DROPLETPHASE: Adding TIMECHECK and general performance improvements.
  • DROPLETPHASE: Extended testing for Dropletphase Subcycling, see COMP_DropletphaseSubcycles.
  • new cv/Ucv Parameter: COMP_TypeSmooth_Rho.
  • Changed default for cv Parameter rel_dist_edge to 0.15 (improves inflow areas in combination with COMP_FillEdges).
  • Changed default for COEFF_mue to 1.0 (previous default was 2.0).
  • Allow soft variables for EVENT definition (analogous to INTEGRATION).
  • SAMG multiphase context: Add a suffix for all scalar MESHFREE contexts to distinguish between v-- and vp-
  • approxY: Add further optional parameter to limit overshoots of the approximation.
  • Common variables prec_seek_holes, prec_act_BND and prec_comp_FS are now deprecated and can be safely removed
  • LSDYNA: Add support for TShell elements (experimental).
  • LSDYNA: Add new NamesOfStopwatches for TIMECHECK (experimental).
  • COMP_MemCheck: Memory information is total memory of the node with the highest workload.
  • ORGANIZE_OppositePoints_Version: Add option to switch off the kob comparison (experimental).
Tickets:
  • #79104: enabling the usage of more UserDefinedIndices.
