GeometryMovement

movement of the STANDBY-pointcloud during simulation

The toMove{} functionality allows the user to let the pointcloud move during time integration. toMove{} assigns a proper MOVE-flag to the pointcloud:
begin_pointcloud{} include{/m/scratch/hive/FPM/JK/results/KarreOriginal.case} format{ensight} ... timeFrame{20} ... variables{ %ind_p%="pressure" , %ind_v(1)%="velocity" , %ind_h%="H" } ... toChamber{11} ... toMaterial{$MatStandby$} ... toMove{$SomeDefinedMoveIndex$} end_pointcloud MOVE($SomeDefinedMoveIndex$) = (%MOVE_...%, ... )
Any movement function as described in MOVE is allowed. AN ALTERNATIVE to the toMove{}-functionality would be:
begin_pointcloud{} include{/m/scratch/hive/FPM/JK/results/KarreOriginal.case} format{ensight} ... timeFrame{20} ... variables{ %ind_p%="pressure" , %ind_v(1)%="velocity" , %ind_h%="H" } ... toChamber{11} ... toMaterial{$MatStandby$} end_pointcloud INITDATA($MatStandbby$, %ind_MOVE% ) = $SomeDefinedMoveIndex$ MOVE($SomeDefinedMoveIndex$) = (%MOVE_...%, ... )
i.e. the toMove{} is simply assigning the $SomeDefinedMoveIndex$ with the appropriate variable %ind_MOVE%
