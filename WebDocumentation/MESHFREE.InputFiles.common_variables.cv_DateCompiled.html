cv_DateCompiled

key to unlock MESHFREE versions that exceeded the compilation date limit

Some MESHFREE versions are not protected by a license key but only by a time limit for the compilation date. In order to unlock a version that has exceeded the limit, a key has to be generated with given seed and provided to the simulation to unlock the respective version. Procedure:
  • adapt "seed" (integer number) and "Str" (current date) in "test_EnDeCoding.f90"
  • compile test_EnDeCoding.f90 by loading the standard modules for compilation of MESHFREE and executing the corresponding compilation script by:
    ./compile_test_DeEnCoding
  • execute the generated program by:
    ./a.out
  • add the parameters cv_DateCompiled and cv_DateCompiled_Seed to the common_variables of your simulation setup and fill them with the output of the previous step ("Str1" and "seed")
  • run the simulation as usual
cv_DateCompiled = "n(>Qo'ohfOJC>7F_u" # Str = 'FPMJUN 05 2020FPM', seed = 40 cv_DateCompiled_Seed = 40
